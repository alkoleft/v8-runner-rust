# Agent Rules for v8-runner

## ОБЯЗАТЕЛЬНЫЕ требования к ревью

Любое ревью изменений в Rust-коде считается неполным без явного применения скилла `/rust-expert-best-practices-code-review`.

1. **Review субагент** при ревью Rust-изменений должен независимо применять checklist и правила из `/rust-expert-best-practices-code-review`, даже если отдельный Rust expert субагент запускается параллельно
2. **Rust expert субагент** остается обязательным независимым контуром проверки перед коммитом и не заменяется обычным reviewer-проходом
3. Если задача включает Rust code review, refactoring, error handling, type safety, API design или performance-sensitive изменения, применение `/rust-expert-best-practices-code-review` обязательно
4. Отсутствие отдельного прогона по этому скиллу считается нарушением требований к ревью; findings из него нельзя молча игнорировать — каждый finding должен быть либо исправлен, либо явно зафиксирован как accepted waiver с кратким обоснованием

## ОБЯЗАТЕЛЬНО в ходе работы

Для нетривиальных задач использовать субагентов не только перед коммитом, но и во время исследования и проверки, если subagent tooling доступен в текущей среде.

1. **Explorer субагент** — для repo-wide поиска по файлам, чтения и сбора evidence, поиска затронутых модулей, API и контрактов
2. **Tests/Tester субагент** — для запуска тестов, smoke-check и других проверок (`cargo test`, targeted suites, integration checks)
3. **Reviewer субагент** — для независимой проверки рисков, архитектурных последствий и полноты изменений, если задача затрагивает несколько модулей, публичные контракты или сложный refactoring
4. Если подзадачи независимы, запускать их параллельно; основной агент должен интегрировать результаты, а не делать все поиски и проверки только локально
5. Если subagent tooling недоступен, sidecar-задача технически не делегируется или параллельный запуск недоступен, основной агент обязан выполнить такую проверку локально и явно зафиксировать этот fallback
6. Для простых однофайловых косметических правок допускается исключение, но для поиска по файлам, тестов, review и других независимых sidecar-задач субагенты считаются обязательными, когда tooling доступен

Цель правила: разгрузить основной поток реализации, ускорить поиск по большому репозиторию и держать review/test evidence отдельным независимым контуром.

## Repo-local SKILL

`v8-runner/SKILL.md` — это скилл для использования `v8-runner` в других 1С-проектах. При реализации задач актуализировать его, если меняются команды, workflow, контракты конфигурации, ограничения или диагностические практики, важные для внешнего применения.

Правки в `v8-runner/SKILL.md` должны быть краткими, емкими и по делу: добавлять только применимые инструкции, без пересказа внутренних деталей реализации и без раздувания справки.
