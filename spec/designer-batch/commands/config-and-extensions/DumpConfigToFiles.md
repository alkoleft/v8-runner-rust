# DumpConfigToFiles

## Назначение

Выгружает конфигурацию или расширения в XML-структуру на диске либо в ZIP-архив. Команда поддерживает полную, инкрементальную и выборочную выгрузку.

## Синтаксис

```text
/DumpConfigToFiles <каталог выгрузки> [-Extension <имя расширения>]
[-AllExtensions] [-format] [-update][-force][-getChanges <имя файла>]
[-configDumpInfoForChanges <имя файла>][-configDumpInfoOnly]
[-Server][-Archive <имя файла>]
[-listFile <имя файла>][-ignoreUnresolvedReferences]
```

```text
1cv8.exe DESIGNER /S "localhost/BaseName" /DumpConfigToFiles c:/xml -Server -JobsCount 5
```

## Параметры

- `<каталог выгрузки>` — каталог назначения для XML-файлов.
- `-Extension <имя расширения>` — выгружает только указанное расширение.
- `-AllExtensions` — выгружает все расширения.
- `-format Hierarchical|Plain` — задает формат выгрузки.
- `-update` — обновляет существующую выгрузку по файлу состояний `ConfigDumpInfo.xml`.
- `-force` — при несовпадении версии формата вместо ошибки выполняет полную выгрузку.
- `-getChanges <имя файла>` — пишет список изменений относительно предыдущей выгрузки.
- `-configDumpInfoForChanges <имя файла>` — использует внешний файл состояний для вычисления изменений.
- `-configDumpInfoOnly` — выгружает только `ConfigDumpInfo.xml`.
- `-Server` — выполняет многопоточную выгрузку на сервере.
- `-Archive <имя файла>` — сохраняет результат в ZIP-архив.
- `-listFile <имя файла>` — ограничивает выгрузку объектами из списка.
- `-ignoreUnresolvedReferences` — не выгружает недостижимые ссылки на удаленные объекты.

## Связи

- По умолчанию `-format` использует `Hierarchical`.
- `Plain` считается устаревшим; для новых сценариев лучше использовать `Hierarchical`.
- `-update` опирается на `ConfigDumpInfo.xml` в каталоге выгрузки.
- `-force` имеет смысл вместе с `-update`; без `-update` он не нужен.
- `-getChanges <имя файла>` можно комбинировать с `-configDumpInfoForChanges <имя файла>`.
- `-configDumpInfoForChanges <имя файла>` используется только вместе с `-update` и `-getChanges <имя файла>`.
- `-configDumpInfoOnly` можно комбинировать с `-format` и `-Archive <имя файла>`.
- `-Server` допускает дополнительный параметр `-JobsCount N`.
- `-Archive <имя файла>` можно использовать как для полной, так и для выборочной выгрузки.
- `-listFile <имя файла>` можно комбинировать с `-Archive <имя файла>`.

## Примечания

- При обычной выгрузке формируется файл состояний `ConfigDumpInfo.xml`.
- `-Extension <имя расширения>` возвращает `0` при успехе и `1`, если расширение не найдено или операция завершилась ошибкой.
- При `-ignoreUnresolvedReferences` платформа меняет или пропускает недостижимые ссылки, поэтому повторная загрузка такой выгрузки может дать отличия в сравнении с исходной конфигурацией.
