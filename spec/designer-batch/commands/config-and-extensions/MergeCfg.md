# MergeCfg

## Назначение

Объединяет текущую конфигурацию с `cf` или `cfe` файлом по файлу настроек.

## Синтаксис

```text
/MergeCfg <имя cf/cfe-файла> -Settings <имя файла настроек>
[-EnableSupport | -DisableSupport][-IncludeObjectsByUnresolvedRefs | -ClearUnresolvedRefs]
[-Extension <имя расширения>][-force]
```

## Параметры

- `<имя cf/cfe-файла>` — файл конфигурации или расширения для объединения.
- `-Settings <имя файла настроек>` — файл настроек объединения.
- `-EnableSupport` — ставит конфигурацию на поддержку при объединении, если это возможно.
- `-DisableSupport` — не ставит конфигурацию на поддержку.
- `-IncludeObjectsByUnresolvedRefs` — включает объекты с неразрешенными ссылками в объединение.
- `-ClearUnresolvedRefs` — очищает неразрешенные ссылки.
- `-Extension <имя расширения>` — объединяет расширение с расширением из `cfe`-файла.
- `-force` — игнорирует предупреждения и продолжает объединение.

## Связи

- `-EnableSupport` и `-DisableSupport` взаимоисключают друг друга.
- `-IncludeObjectsByUnresolvedRefs` и `-ClearUnresolvedRefs` задают альтернативное поведение для неразрешенных ссылок.
- Если `-Extension <имя расширения>` указан вместе с `-EnableSupport` или `-DisableSupport`, объединение завершается ошибкой.
- Без `-force` объединение прерывается на предупреждениях о настройках и удаляемых объектах со ссылками.

## Примечания

- Правила поддержки для `-EnableSupport` должны быть заданы в файле настроек.
- Все каталоги в полном пути к входному файлу и `-Settings` должны существовать заранее.
