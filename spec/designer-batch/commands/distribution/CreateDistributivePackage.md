# CreateDistributivePackage

## Назначение

Создает ZIP-архив комплекта поставки и при необходимости дополнительные файлы по готовому описанию комплекта.

## Синтаксис

```text
/CreateDistributivePackage <имя каталога> -File <имя файла описания комплекта поставки>
-PackageFileName <имя архива> [-Option <вариант поставки>] [-MakeSetup] [-MakeFiles]
[-digisign <имя файла с параметрами лицензирования>] [-WarningAsError]
```

## Параметры

- `<имя каталога>` — каталог для результата.
- `-File <имя файла описания комплекта поставки>` — описание комплекта поставки.
- `-PackageFileName <имя архива>` — имя архива комплекта поставки.
- `-Option <вариант поставки>` — имя варианта поставки из описания.
- `-MakeSetup` — создает комплект поставки.
- `-MakeFiles` — создает только файлы комплекта поставки.
- `-digisign <имя файла с параметрами лицензирования>` — параметры лицензирования рабочего места.
- `-WarningAsError` — трактует предупреждения как ошибки.

## Связи

- `-MakeSetup` и `-MakeFiles` взаимоисключают друг друга.
- Если ни `-MakeSetup`, ни `-MakeFiles` не указаны, используется `-MakeSetup`.
- `-PackageFileName <имя архива>` имеет смысл только вместе с `-MakeSetup`.
- Если `-PackageFileName <имя архива>` задан без расширения `.zip`, платформа добавит его автоматически.
- Если `-Option <вариант поставки>` не указан, используется вариант `Полный`.
- `-WarningAsError` делает несоответствие подписи и метаданных фатальной ошибкой.

## Примечания

- Если имя архива не задано явно, используется `updsetup.zip`.
- Все каталоги в полном пути к входным и выходным файлам должны существовать заранее.
