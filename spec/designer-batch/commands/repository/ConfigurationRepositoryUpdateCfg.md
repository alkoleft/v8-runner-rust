# ConfigurationRepositoryUpdateCfg

## Назначение

Обновляет конфигурацию из хранилища конфигурации.

## Синтаксис

```text
/ConfigurationRepositoryUpdateCfg [-Extension <имя расширения>]
[-v <номер версии хранилища>] [-revised] [-force]
[-objects <имя файла со списком объектов>]
```

## Параметры

- `-Extension <имя расширения>` — обновляет указанное расширение вместо основной конфигурации.
- `-v <номер версии хранилища>` — номер версии для получения.
- `-revised` — получает захваченные объекты, если это потребуется.
- `-force` — подтверждает получение новых объектов и удаление существующих.
- `-objects <имя файла со списком объектов>` — обновляет только объекты из XML-списка.

## Связи

- Если конфигурация уже подключена к хранилищу, `-v <номер версии хранилища>` игнорируется и берется актуальная версия.
- Если конфигурация не подключена к хранилищу, `-v <номер версии хранилища>` позволяет получить конкретную версию; `-1` или отсутствие номера означает актуальную версию.
- `-revised` игнорируется, если конфигурация не подключена к хранилищу.
- Если `-objects <имя файла со списком объектов>` не указан, обновляется вся конфигурация.

## Примечания

- Все каталоги в полном пути к файлу `-objects` должны существовать заранее.
