# ConfigurationRepositoryUnlock

## Назначение

Снимает захват объектов в хранилище конфигурации.

## Синтаксис

```text
/ConfigurationRepositoryUnlock [-Extension <имя расширения>]
[-objects <имя файла со списком объектов>] [-force]
```

## Параметры

- `-Extension <имя расширения>` — работает с хранилищем указанного расширения.
- `-objects <имя файла со списком объектов>` — снимает захват только для объектов из XML-списка.
- `-force` — отбрасывает локальные изменения и получает объекты из хранилища.

## Связи

- Если `-objects <имя файла со списком объектов>` не указан, захват снимается для всех объектов конфигурации.
- Без `-force` команда завершится ошибкой, если есть локально измененные объекты.
- Если `-Extension <имя расширения>` не указан, команда работает с основной конфигурацией.

## Примечания

- Объекты, не захваченные текущим пользователем или захваченные другим пользователем, не считаются ошибкой в списке `-objects`.
- Все каталоги в полном пути к файлу `-objects` должны существовать заранее.
