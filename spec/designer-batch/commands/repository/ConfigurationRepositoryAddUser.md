# ConfigurationRepositoryAddUser

## Назначение

Создание пользователя хранилища конфигурации. Пользователь, от имени которого выполняется подключение к хранилищу, должен обладать административными правами. Если пользователь с указанным именем существует, то пользователь добавлен не будет.

## Синтаксис

```text
/ConfigurationRepositoryAddUser [-Extension <имя расширения>] -User <Имя> -Pwd <Пароль> -Rights <Права> [-RestoreDeletedUser]
```

## Параметры

- `-Extension <имя расширения>` — имя расширения. Если параметр не указан, выполняется попытка соединения с хранилищем основной конфигурации, и команда выполняется для основной конфигурации. Если параметр указан, выполняется попытка соединения с хранилищем указанного расширения, и команда выполняется для этого хранилища.
- `-User <Имя>` — имя создаваемого пользователя.
- `-Pwd <Пароль>` — пароль создаваемого пользователя. Если при создании хранилища для `/ConfigurationRepositoryCreate` был указан параметр `CheckPasswordComplexity`, то при создании пользователя будет выполнена проверка пароля на соответствие политике.
- `-Rights <Права>` — права пользователя. Возможные значения:
- `ReadOnly` — право на просмотр,
- `LockObjects` — право на захват объектов,
- `ManageConfigurationVersions` — право на изменение состава версий,
- `Administration` — право на административные функции.
- `-RestoreDeletedUser` — если обнаружен удаленный пользователь с таким же именем, он будет восстановлен.
