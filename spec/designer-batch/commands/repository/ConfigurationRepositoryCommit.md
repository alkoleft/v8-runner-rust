# ConfigurationRepositoryCommit

## Назначение

Помещение изменений объектов в хранилище конфигурации.

## Синтаксис

```text
/ConfigurationRepositoryCommit [-objects <имя файла со списком объектов>]
[-comment "<Текст комментария>"] [-keepLocked] [-force] [-Extension <имя расширения>]
```

## Параметры

- `-objects <имя файла со списком объектов>` — путь к файлу формата XML со списком объектов. При наличии в списке объектов, не захваченных текущим пользователем или захваченных другим пользователем, ошибка выдана не будет. Подробнее о формате файла см в документации.
- `-comment "<Текст комментария>"` — комментарий. Чтобы установить многострочный комментарий, для каждой строки следует использовать свой параметр `-comment`.
- `-keepLocked` — оставлять захват для помещенных объектов.
- `-force` — если параметр используется, при обнаружении ссылок на удаленные объекты будет выполнена попытка их очистить.
- `-Extension <имя расширения>` — имя расширения. Если параметр не указан, выполняется попытка соединения с хранилищем основной конфигурации, и команда выполняется для основной конфигурации. Если параметр указан, выполняется попытка соединения с хранилищем указанного расширения, и команда выполняется для этого хранилища.

## Связи

- `-objects <имя файла со списком объектов>`: если параметр используется, будет выполнена попытка поместить только объекты, указанные в файле.
- `-objects <имя файла со списком объектов>`: если параметр не используется, будут помещены изменения всех объектов конфигурации.

## Примечания

- `-force`: если параметр не указан, при обнаружении ссылок на удаленные объекты будет выдана ошибка.
