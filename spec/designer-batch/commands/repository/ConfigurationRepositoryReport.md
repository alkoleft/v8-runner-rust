# ConfigurationRepositoryReport

## Синтаксис

```text
/ConfigurationRepositoryReport <имя файла> [-Extension <имя расширения>]
[-NBegin <номер версии>] [-NEnd <номер версии>] [-DateBegin] [-DateEnd] [-GroupByObject]
[-GroupByComment] [-DoNotIncludeVersionsWithLabels] [-IncludeOnlyVersionsWithLabels]
[-IncludeCommentLinesWithDoubleSlash] [-ConfigurationVersion <версия конфигурации>]
[-ReportFormat < txt | mxl>]
```

## Описание

**— **построение отчета по истории хранилища. Если параметры группировки не указаны и режим совместимости указан "**Не используется**", то отчет формируется с группировкой по версиям. В режимах совместимости "**Версия 8.1**" и "**Версия 8.2.13**" отчет формируется с группировкой по объектам. Если конфигурация базы данных отличается от редактируемой по свойству совместимости, при обработке командной строки учитывается значение режима совместимости конфигурации базы данных.

### Параметры

- **<имя файла>** — имя файла, в который выводится отчет;

- **-Extension <имя расширения>** — имя расширения. Если параметр не указан, выполняется попытка соединения с хранилищем основной конфигурации, и команда выполняется для основной конфигурации. Если параметр указан, выполняется попытка соединения с хранилищем указанного расширения, и команда выполняется для этого хранилища;

- **-NBegin** — номер сохраненной версии, от которой начинается строиться отчет. Если указано значение "-1", отчет строится для последней версии в хранилище;

- **-NEnd** — номер сохраненной версии, по которую строится отчет;

- **-DateBegin** — дата, начиная с которой строится отчет. Для формирования даты используется синтаксис строки, передаваемой конструктору типа **Дата** встроенного языка.****.
Например:
'2017\03\23 10~45~25'
'2017.03.23 10:45:25'
'20170323104525';

- **-DateEnd** — дата, по которую строится отчет. Для формирования даты используется синтаксис строки, передаваемой конструктору типа **Дата** встроенного языка.****;

- **-GroupByObject** — признак формирования отчета по версиям с группировкой по объектам;

- **-GroupByComment** — признак формирования отчета по версиям с группировкой по комментарию;

- **-DoNotIncludeVersionsWithLabels** — если установлен, то версии с метками в отчет включены не будут;

- **-IncludeOnlyVersionsWithLabels** — если установлен, то в отчет будут включены только версии с метками.
Игнорируется, если используется параметр **-DoNotIncludeVersionsWithLabels**;

- **-IncludeCommentLinesWithDoubleSlash** — признак вовда строк комментариев, которые начинаются с символа "//";

- **-ConfigurationVersion <версия конфигурации>** — версия конфигурации, для которой выводятся версии хранилища;

- **-ReportFormat < txt | mxl>** — формат файла отчета:  текстовый документ  (txt) или табличный документ (mxl).
По умолчанию - mxl.

**Примеры**:

для конфигурации, не присоединенной к текущему хранилищу:

DESIGNER /F"D:\V8\Cfgs82\ИБ82" /ConfigurationRepositoryF "D:\V8\Cfgs82" /ConfigurationRepositoryN "Администратор" /ConfigurationRepositoryReport "D:\ByObject.mxl" -NBegin 1 -NEnd 2 –GroupByObject

для присоединенной к хранилищу конфигурации, информация для отчетов берется из текущего хранилища, версии с метками не включаются в отчет:

DESIGNER /F"D:\V8\Cfgs82\ИБ82" /ConfigurationRepositoryReport "D:\ByComment.mxl" -NBegin 1 -NEnd 2 -GroupByComment -DoNotIncludeVersionsWithLabels
